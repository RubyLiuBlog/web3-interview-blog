12:T6b6,<h1>Next.js Dapp 应用中钱包登录后的 Token 存储方案</h1>
<h2>客户端存储选项</h2>
<ol>
<li>
<p><strong>localStorage</strong></p>
<ul>
<li><strong>适用场景</strong>：客户端组件中</li>
<li><strong>优点</strong>：简单实现，持久化存储</li>
<li><strong>缺点</strong>：仅浏览器端可用，不安全，XSS 风险</li>
</ul>
</li>
<li>
<p><strong>Cookie</strong></p>
<ul>
<li><strong>适用场景</strong>：需要在客户端/服务端共享的场景</li>
<li><strong>优点</strong>：可设置 HttpOnly 提高安全性</li>
<li><strong>缺点</strong>：每次请求都会发送，大小限制</li>
</ul>
</li>
<li>
<p><strong>内存状态管理</strong></p>
<ul>
<li><strong>工具</strong>：React Context、Zustand、Redux 等</li>
<li><strong>优点</strong>：访问快速，适合会话期间使用</li>
<li><strong>缺点</strong>：页面刷新后丢失</li>
</ul>
</li>
</ol>
<h2>最佳实践</h2>
<h3>推荐方案</h3>
<ul>
<li>
<p><strong>短期 Token</strong>：使用 HttpOnly Cookie 存储</p>
<ul>
<li>防 XSS 攻击</li>
<li>服务端可访问</li>
<li>可设置适当过期时间</li>
</ul>
</li>
<li>
<p><strong>长期会话</strong>：结合使用</p>
<ol>
<li>短期访问 Token 存 Cookie（HttpOnly）</li>
<li>刷新 Token 存储在更安全的机制中</li>
</ol>
</li>
</ul>
<h3>实现考虑</h3>
<ul>
<li>客户端组件中使用"use client"</li>
<li>考虑使用 Next.js 的 middleware 进行 token 验证</li>
<li>实现自动刷新 token 机制</li>
</ul>
<h3>注意事项</h3>
<ul>
<li>避免在 localStorage 中存储敏感 token</li>
<li>考虑使用加密技术增强安全性</li>
<li>实现适当的 token 过期和刷新逻辑</li>
</ul>
