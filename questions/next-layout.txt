12:Ta1b,<h1>Next.js App Router 中 Layout 的最佳实践</h1>
<h2>架构设计</h2>
<ol>
<li>
<p><strong>布局分层</strong></p>
<ul>
<li>根布局：全局 UI 元素、主题配置</li>
<li>功能布局：按功能区域划分（认证区域、管理区域）</li>
<li>页面布局：特定页面组结构</li>
</ul>
</li>
<li>
<p><strong>职责划分</strong></p>
<ul>
<li>每个布局职责单一</li>
<li>避免过深嵌套（通常不超过 3-4 层）</li>
<li>将共享状态放在适当的布局层级</li>
</ul>
</li>
</ol>
<h2>性能优化</h2>
<ol>
<li>
<p><strong>组件分割</strong></p>
<ul>
<li>将布局拆分为更小的组件</li>
<li>避免大型单体布局文件</li>
<li>利用 React.memo 减少不必要渲染</li>
</ul>
</li>
<li>
<p><strong>数据获取</strong></p>
<ul>
<li>在合适的布局层级获取数据</li>
<li>利用平行数据获取</li>
<li>使用缓存和重验证策略</li>
</ul>
</li>
<li>
<p><strong>加载策略</strong></p>
<ul>
<li>实现 loading.js 提供加载状态</li>
<li>考虑使用 Suspense 边界细分加载区域</li>
<li>为关键布局元素预先加载数据</li>
</ul>
</li>
</ol>
<h2>状态管理</h2>
<ol>
<li>
<p><strong>客户端 vs 服务端</strong></p>
<ul>
<li>默认保持为服务端组件</li>
<li>只在需要交互时添加"use client"</li>
<li>将客户端逻辑隔离到子组件</li>
</ul>
</li>
<li>
<p><strong>共享状态</strong></p>
<ul>
<li>Context Provider 放置在适当布局层级</li>
<li>考虑使用第三方状态库（Zustand 等）</li>
<li>避免过度共享导致不必要渲染</li>
</ul>
</li>
</ol>
<h2>实用建议</h2>
<ol>
<li>
<p><strong>路由组织</strong></p>
<ul>
<li>使用 Route Groups <code>(folder)</code>隔离布局</li>
<li>利用<code>layout.js</code>和<code>template.js</code>的区别</li>
<li>善用<code>not-found.js</code>和<code>error.js</code></li>
</ul>
</li>
<li>
<p><strong>跨布局通信</strong></p>
<ul>
<li>使用 URL 参数传递简单状态</li>
<li>优先考虑服务端状态而非客户端状态</li>
<li>需要时使用 localStorage/cookie 存储持久数据</li>
</ul>
</li>
<li>
<p><strong>响应式设计</strong></p>
<ul>
<li>布局组件使用响应式设计</li>
<li>考虑使用 CSS Grid 或 Flexbox</li>
<li>使用<code>useMediaQuery</code>或 CSS 媒体查询适配不同设备</li>
</ul>
</li>
<li>
<p><strong>代码组织</strong></p>
<ul>
<li>将复杂布局提取到<code>/components</code>目录</li>
<li>使用<code>/lib</code>存放布局相关工具函数</li>
<li>布局样式使用模块化 CSS 或样式库</li>
</ul>
</li>
</ol>
